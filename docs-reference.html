Sidebar structure

The left rail is now driven by a shared configuration in the main script. Each group defines a label and a set of pages, which keeps navigation stable even when the content headings change.

Group labels

Use short, durable labels such as Guide, Reference, or API. These should describe clusters of pages rather than temporary article topics.

Layout breakpoints

The docs shell now behaves in three layers: a stacked mobile layout, a two-column desktop layout with the left sidebar, and a three-column wide layout that restores the right-hand table of contents.

Desktop behavior

  • The left sidebar becomes sticky at large breakpoints.
  • The right TOC returns only on wider screens.
  • The active page remains highlighted while section links track scroll.

Theme hooks

The sidebar and TOC both inherit the existing color tokens, which keeps light and dark mode behavior centralized in the same CSS custom properties already used across the site.
